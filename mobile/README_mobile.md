# 걸음지기 모바일 웹앱 (폰 테스트)

폰 브라우저에서 **카메라 + 온디바이스 AI + 한국어 음성**으로 도는 보행 보조 프로토타입.
앱 설치 불필요. 카메라는 **HTTPS(또는 localhost)** 에서만 열리므로 아래 방법 중 하나로 접속.

> ⚠️ 보조 정보 제공용입니다. 흰지팡이·안내견·보행훈련을 대체하지 않으며 사고 방지를 보장하지 않습니다.

## 두 가지 인식 모드
| 모드 | 엔진 | 잡는 것 | 속도 |
|---|---|---|---|
| **빠름** | COCO-SSD | 차·오토바이·사람·자전거·버스/트럭 + 벤치·화분·소화전·표지판 등 | 빠름(폰에서 수 fps) |
| **정밀 보행로** | DeepLab Cityscapes | **보도/도로(걸을 수 있는 길)** + **전봇대**·차·사람 + 경로 침범물(쓰레기통 등 보행불가 영역) | 느림(1~3 fps) |

- "정밀 보행로"는 **걸을 수 있는 길만** 코리도로 보고, 그 안에 보행 불가한 것이 들어오면 `앞 장애물`/`보도 끝 주의`로 안내 → 전봇대·쓰레기통처럼 이름표가 없는 것도 잡음.
- 음성 우선순위 L1~L4 · 쿨다운 · 인터럽트 · 정지침묵은 데스크톱 로직과 동일(`walk_logic.js`).

## 내비게이션(방향 안내) — 논문 반영
`내비 ON` 버튼(정밀 보행로 모드에서 동작). AI Guide Dog 논문(arXiv 2501.07957)의 **전/좌/우 경로 예측**을, 우리가 이미 돌리는 **보행가능영역(sidewalk/road) 세그멘테이션**에서 기하학적으로 유도해 `직진`·`왼쪽으로`·`오른쪽으로`·`정지, 길 막힘`을 안내합니다.
- **방향이 바뀔 때만** 말함(연속 잔소리 금지), 히스테리시스로 흔들림 억제.
- **위험 경고가 있으면 내비는 침묵**(안전 우선). 파란 배너로 위험(빨강/주황)과 구분.
- 학습 데이터 불필요 · 온디바이스 동작. 논문의 2fps·보행속도 0.72m/s 근거와 정합.

## PC에서 먼저 테스트 (폰·카메라·방화벽 없이)
`동영상` 버튼 → 보행 영상 파일 선택 → 카메라 대신 그 영상으로 파이프라인이 돌아갑니다.
`localhost`는 보안 컨텍스트라 PC 브라우저에서 바로 됩니다.
```bash
python -m http.server 8777 --directory .
# → PC 브라우저: http://localhost:8777  → '동영상' → 걷는 영상 선택
#   (정밀 보행로 + 내비 ON 으로 방향 안내 확인)
```
`자가검사` 버튼으로 위험/내비 로직 7종 즉시 확인 가능.

## 폰에 올리는 3가지 방법

### A. 같은 Wi-Fi + 로컬 HTTPS (가장 빠름, 추천)
```bash
pip install cryptography
python serve_https.py
# → 표시되는 https://<PC-IP>:8443 을 폰 브라우저에 입력
# → 인증서 경고: '고급 → 계속 진행' 허용 → 시작
```

### B. GitHub Pages (공개 HTTPS, 링크 공유)
저장소를 GitHub에 올리고 Pages를 켜면 `https://<계정>.github.io/걸음지기/mobile/` 로 접속.
※ 공개 게시이므로 종현님이 직접 push/설정.

### C. 정적 호스팅 (Netlify/Vercel drop 등)
`mobile/` 폴더를 드래그 업로드 → HTTPS 링크 발급.

## 폰별 참고
- **Android Chrome**: 후면카메라·한국어 음성 대체로 잘 됨.
- **iOS Safari**: 음성은 사용자가 '시작'을 눌러야 잠금 해제(코드 반영됨). 자체서명 인증서는 한 번 신뢰 필요.
- 첫 실행 시 CDN에서 모델을 받으므로 인터넷 필요(이후 캐시). 음성 합성 자체는 오프라인 동작.

## 화면
- 상단: fps / 모드 / 보행·정지 상태
- 파란 사다리꼴 = 전방 보행 코리도(집중 감시 영역)
- 박스 색: 빨강=가까움, 주황=중간, 초록=멀리
- 하단 큰 배너 + 음성 + L1 진동
- '자가검사' 버튼: 포팅 로직 정상 여부 즉시 확인
